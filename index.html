HTML Lesson 1: The Power of Semantic Tags

**Semantic** means **"relating to meaning in language or logic."** In HTML, semantic elements tell the browser, developer, and search engines *what* the content is, not just *how* it should look.

Why Use Semantic Tags?

  • **Accessibility (A11y):** Screen readers rely on tags like <nav> to guide users.
  • **SEO:** Search engines prioritize content wrapped in meaningful tags like <article>.
  • **Readability:** Semantic code is clean and easy for team members to understand.

Code Example: Semantic vs. Non-Semantic


<div id="header">
    <div class="nav-menu">...</div>
</div>
<div id="content-main">
    <h1>Post Title</h1>
</div>
<div id="footer">Copyright</div>


<header>
    <h1>My Site</h1>
    <nav>...</nav>
</header>

<main>
    <article>
        <h2>Semantic HTML</h2>
        <p>Article content.</p>
    </article>
</main>

<footer>
    <p>&copy; 2025</p>
</footer>

CSS Lesson 1: Mastering The Box Model

Every HTML element is a **rectangular box**. The Box Model is the rulebook that defines how these boxes interact with one another by controlling their size and space.

The Box Model has four nested layers, from the inside out:

  1. **Content:** The actual data (text, image). Controlled by `width`/`height`.
  2. **Padding:** **Internal space** between the content and the border. Takes the element's background color.
  3. **Border:** The line wrapped around the padding.
  4. **Margin:** **External space** outside the border. Creates space between elements. Always transparent.

Code Example: Box Properties


<div class="box">This is the Content Area.</div>


.box {
    /* 1. Content */
    width: 300px; 
    height: 100px;
    background-color: #4a4a75;

    /* 2. Padding (Internal) */
    padding: 20px; 

    /* 3. Border */
    border: 5px solid red;

    /* 4. Margin (External) */
    margin: 40px auto; /* Centers horizontally and adds vertical space */

    /* Total rendered width will be 350px (300 + 20 + 20 + 5 + 5) without border-box! */
}

Pro Standard: box-sizing: border-box

By default, adding padding/border increases the total size of the element (Content-Box Model). The professional standard is to use the **Border-Box Model**, which includes padding and border *inside* the defined `width`/`height`. 

* {
    box-sizing: border-box; 
    /* USE THIS IN EVERY PROJECT! Padding and border are now calculated INWARD. */
}

CSS Lesson 2: Introduction to Flexbox

**Flexbox** (Flexible Box Module) is the modern, powerful solution for **one-dimensional layout** (aligning items along a single line, either horizontally or vertically).

The Container/Item Model

Flexbox works by creating a relationship between a **parent element (the Container)** and its **direct children (the Items)**.

  • **Flex Container:** Set display: flex; on the parent.
  • **Flex Items:** The children inside the container are automatically adjusted.

Code Example: Creating a Centered Menu


<nav class="flex-container">
    <a>Home</a>
    <a>About</a>
    <a>Contact</a>
</nav>


.flex-container {
    display: flex; /* 1. ACTIVATES Flexbox (sets main axis to horizontal) */
    height: 100px;
    background-color: #3f3f5a;
    border-radius: 0.5rem;

    /* 2. Horizontal Positioning (Main Axis) */
    justify-content: center; 

    /* 3. Vertical Positioning (Cross Axis) */
    align-items: center; 
    
    /* The items (the <a> tags) are now perfectly centered both ways! */
}
